# App Store screenshots

Marketing screenshots for the App Store, composed from **real app captures** plus
a branded background + caption. Generated by
[`scripts/frame-app-store-screenshots.mjs`](../../scripts/frame-app-store-screenshots.mjs)
(config: [`scripts/store-screenshots.config.mjs`](../../scripts/store-screenshots.config.mjs)).

See the `frame-app-store-screenshots` skill for the full workflow.

## Quick use

1. Capture the real app screens (one PNG per shot, per locale) and drop them in:

   ```
   raw/<locale>/<name>.png      e.g. raw/en-US/01-home.png, raw/cs/01-home.png
   ```

   The filenames must match the `raw` values in `store-screenshots.config.mjs`.
   Capture on an **iPhone 16/17 Pro Max** simulator (so the source is 1320×2868)
   using the seeded reviewer account so charts/calendar aren't empty.

2. Check which captures are present:

   ```bash
   npm run frame-screenshots -- --check
   ```

3. Render the framed, exact-size images:

   ```bash
   npm run frame-screenshots
   ```

   Output lands in `framed/<locale>/<device>/NN_<name>.png`, sized exactly for
   App Store Connect (6.9" = 1320×2868, 6.7" = 1290×2796).

4. Upload `framed/**` to ASC (manually, or wire it into `deliver` later).

`raw/` and `framed/` are git-ignored — they're inputs/outputs, not source.

> **Apple Guideline 2.3.3:** screenshots must depict the real app. This tool
> only adds marketing chrome around your genuine captures — never fabricate UI.
