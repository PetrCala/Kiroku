Sessions Calendar — "how do I know what month I'm on?"

All four variants share the common base: native scrollbar (thin track on the right), equal vertical and horizontal spacing between day cells, and small visual separation between months. The variants differ in how they tell the user which month/year they're currently looking at. Scroll inside each phone frame to feel each option. The recommendation is marked.

Recommendation

A — Inline section labels + sticky overflow

Each month opens with a row label. As you scroll past one, it sticks to the top until the next one arrives. Apple Photos / Notion pattern.

Best-practice rec is A. It's the iOS/Android system pattern (Photos, Messages, Health) for time-series scroll surfaces — inline labels naturally separate months (your point 4), the sticky-overflow keeps the current month visible whenever the user pauses scrolling, and it adds zero permanent chrome to the screen. B is the simplest alternative — clearer but eats one extra row of vertical space and skips visual month boundaries. C is closest to your wheel idea — works, but re-introduces the side division we wanted to remove and the columns are visually crowded. D looks elegant but is invisible when idle, so a returning user has to scroll briefly to confirm where they are.
